3. 難易度・ディストラクタ品質

評価項目評価所見
問題形式優全問「業界シナリオ + 4選択肢」。公式ガイド Sample問題と同形式。単なる暗記でなくトレードオフ判断を問う設計（例: 一時的モデル切替 vs 共有設定変更）。

解説の深さ優各選択肢に加え "Overall explanation" で設計思想・公式docリンク・誤答の陥穽を3段落で解説。復習教材として単体で完結する。
難易度本番相当〜やや上実務経験者（MQC）向け。新モデル機構（effort, adaptive thinking）など最新仕様を取り込んでおり鮮度が高い。81%正答なら本番720ラインは概ね安心圏。

4. 改善点・注意点

  • D4 Eval が1問のみ（1.9%） — 重み2.6%相応だが、_evals/judge/tracing/debugging_ の演習量としては薄い。公式ガイド Sample や Prep M4 S02–S07 で補完推奨。
  • D3 Claude Code が2問のみ — 重み3.1%相応。CLAUDE.md・settings.json・hooks は出題済みだが headless/auto モードは未出題。Prep M3 S02 で補強。
  • 「複数正解の2つ目」漏れが目立つ — あなたの誤問10問中5問（Q5/15/34/41/50）が「1つは正解・もう1つを漏選」。教材の欠陥ではなく受験スキルの問題（→ 弱点分析HTML で対策）。
  • 技術的鮮度のリスク — Sonnet 5 / Opus 4.8 / Haiku 4.5 / effort / compact_20260112 など2026年7月時点の最新仕様を反映。モデルIDや仕様が本番前に更新された場合は要追随（同問題集 Last updated July 2026）。
